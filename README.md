# 📊 Swiggy Sales Data Analysis

## 🚀 Project Overview

The Swiggy Sales Data Analysis project focuses on analyzing food delivery sales data to uncover meaningful business insights. 

This project demonstrates data cleaning, exploratory data analysis (EDA), feature engineering, and data visualization skills using Python.

The objective is to identify revenue trends, top-performing dishes, seasonal patterns, and category-wise sales performance.

---

##  Dataset Description

The dataset includes:

- Order Date
- Dish Name
- Price (INR)
- Other sales-related attributes

Data preprocessing was performed to handle:
- Missing values
- Date formatting issues
- Text standardization
- Feature extraction

## 🔎 Project Workflow

### 1️⃣ Data Cleaning
- Removed missing values
- Standardized column names
- Converted Order Date into datetime format
- Extracted Quarter and Day Name

### 2️⃣ Feature Engineering
- Categorized dishes into **Veg / Non-Veg** using keyword mapping
- Created quarterly revenue grouping
- Generated summary statistics

### 3️⃣ Exploratory Data Analysis (EDA)
- Quarterly revenue trend analysis
- Category-wise revenue comparison
- Top selling dishes identification
- Correlation analysis using heatmap

### 4️⃣ Data Visualization
- Line charts for revenue trends
- Bar charts for category performance
- Heatmaps for correlation analysis
- Interactive visualizations using Plotly

- Observed seasonal revenue trends
- Determined peak ordering periods
- Compared Veg vs Non-Veg revenue contribution
